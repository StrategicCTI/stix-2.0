A
AddressAbstract

<p>The AddressAbstractType class characterizes geographic address information. It is an abstract class, so it MUST be extended via a subclass to capture an address. STIX defines the CIQAddress3.0InstanceType subclass as a default extension to leverage the OASIS Customer Information Quality (CIQ) data model, which is a set of XML specifications for representing characteristic information about individuals and organizations. Details are provided in the extensions package.</p>

Identity

A kind of DescriptiveProperties and IDableConstruct. An instance may have the following properties:

  • Name at most one value of BasicString
  • Role any number of values of IdentityRole
  • Specification at most one instance of DetailedIdentitySpecification

<p>The IdentityType class characterizes identity information for both individuals and organizations. The IdentityType class is intended to be extended via a subclass to capture the structured descriptions of identity information for both individuals and organizations. STIX defines a default extension to the IdentityType class to leverage the OASIS Customer Information Quality (CIQ) data model, which is a set of XML specifications for representing characteristic information about individuals and organizations (see extensions package).</p>

M
MarkingDefinition

A valid instance satisfies the following necessary conditions:

  • id exactly one value of Identifier
  • Marking_Type exactly one value of BasicString

<p>The MarkingStructureType class characterizes the marking information to be applied to STIX content. The class is simply a mechanism for leveraging externally defined marking systems, and it is intended to be extended to enable the expression of any structured or unstructured data marking mechanism. </p>
